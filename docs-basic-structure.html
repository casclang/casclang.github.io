Basic Casclang file structure

Namespace {

System

}

System.Method.call {

Main()

}

System.Method.void {

Main()

{

...

}

Here we can see the basic structure of a Casclang file.

It's important to break it down into chunks, so let's do that:

Namespace {

System

}

Here we can see the first statement called in the code, the "Namespace" method.

Invoking Namespaces in your code give it extended functionality. The "main" namespace that every program will utilise is the "System" namespace. This gives it functionality such as reading user input, creating variables.

Each program method will have a Namespace attached to it, such as the "System.Variable" method. The documentation for each of these methods will inform you of which Namespace it utilises.

• You can read the Namespaces documentation for more information.

System.Method.call {

Main()

}

Here we have another vital part of the structure, this here calls the Main method (also known in other languages as Functions) which is about to be written. Note that it is invoking the "System" namespace that was added to our codebase earlier.

System.Method.void {

Main()

{

...

}

Finally we actually create our Main method. Please note that here we use "void" because it is not returning a value to another method. To learn more about methods read here.

The "..." is used as a placeholder to represent where your code may go.

Finally we close our main method with a closing brace "}"
